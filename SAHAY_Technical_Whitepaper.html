Executive Summary

SAHAY is a next-generation Natural Language Intelligence layer designed specifically for the Matrix COSEC ecosystem. It abstracts complex SQL querying into a simple conversational interface, allowing non-technical stakeholders to perform deep analytical audits on attendance, canteen, and security traffic in real-time.

1. Problem Statement: The Data Bottleneck

In modern enterprise environments, Matrix COSEC serves as a mission-critical repository for attendance and security data. However, organizations frequently face the "Reporting Lag"—where extracting specific, ad-hoc insights (e.g., "Which floor had the most unauthorized breaks last week?") requires complex SQL expertise or hours of manual Excel manipulation. This dependency creates a bottleneck that prevents HR and Security managers from making real-time, data-driven decisions.

2. Solution Approach: The Natural Language Bridge

SAHAY resolves this bottleneck by implementing a Zero-Trust Natural Language to SQL (NL2SQL) Bridge. Our approach utilizes the following core pillars:

  • Neural Translation: Leveraging the 2026 Llama 4 Scout model to translate conversational English into high-precision T-SQL.
  • Schema Decoupling: Using pre-optimized Database Views to ensure the AI only interacts with safe, relevant data points.
  • Deterministic Validation: A hard-coded SQL Validator that audits every AI-generated query before execution, ensuring a 100% Read-Only environment.

3. Key Outcomes and Impact

90% Time Reduction Insights that previously took hours of manual reporting are now delivered in under 2 seconds via conversational query.
Democratic Data Access Managers can query the system directly without needing IT or Database Administrator (DBA) intervention.
Zero-Trust Security The system is architected to be incapable of modifying database records, providing a safe sandbox for data exploration.
Operational Precision Real-time tracking of canteen subsidies and attendance anomalies leads to immediate cost savings and compliance enforcement.

1. The Experience Layer Built using React and Tailwind CSS, this layer provides a glassmorphic, low-latency UI. It handles state management for chat history and real-time data table rendering.
2. The Intelligence Layer A Node.js backend acting as a "Translator." It uses a custom Context Engine (nl2sql-skill) to inject COSEC-specific schema knowledge into every LLM request.

4. The Persistence Layer The local MS SQL Server instance. All queries are executed against pre-optimized Views (Attendance, Canteen, VMS) to ensure data integrity and query performance.

Data Integrity & Security

To ensure enterprise safety, the system implements a **Dual-Verification Protocol**:

  • Protocol A (SQL Validator): Every generated string is parsed for destructive keywords (`DROP`, `TRUNCATE`, `DELETE`, `UPDATE`). Non-compliant strings are instantly blocked.
  • Protocol B (Schema Restriction): The system is instructed to only use approved `Mx_VEW_` views, preventing the LLM from accessing sensitive system tables.

Failover & Redundancy

The 2026 architecture introduces Tiered Model Failover. In the event of a cloud rate-limit or model decommission, the middleware automatically pivots through the pool (Llama 3.3 -> Llama 4 -> Qwen 3 -> Llama 8B) in under 500ms, ensuring zero downtime for critical audits.
